Branches (Ramificações)

Quando copiamos um projeto remoto com o "git clone", é uma boa prática criar em nossa máquina uma nova branch ou seja, uma nova ramificação de desenvolvimento que não afetará a ramificação principal. Isso serve para permitir que façamos alterações à vontade, guardemos no repositório remoto, mas sem necessariamente mexermos na linha de desenvolvimento principal (branch master) que poderíamos dizer que é a oficial num projeto.

a) git checkout -b nomeDaBranch: cria uma nova branch no repositório local.
b) git branch --list: lista todas as branches existentes
c) git push origin nomebranch: Envia a nova branch para o repositório remoto.

Quando fazemos o procedimento acima, estamos automaticamente numa branch nova. Faça as modificações necessárias, adicione-os e comite. Para que eles sejam enviados para a branch correta, use o comando [git push https://github.com/nomegit/repo nomeDaBranch]

Merge:

Para fazer merge, é preciso estar na branch que vai receber atualizações de outra ramificação de desenvolvimento. Basta utilizar o comando abaixo, não esquecendo de dar push no final. O comando abaixo realiza um merge automático de todos os arquivos diferentes entre as ramificações.
g) git merge nomeDaOutraBranch

Deletando branches:

De qualquer branch, você pode deletar uma ramificação, bastando digitar os comandos abaixo de acordo com a necessidade:
a) git branch -d nomeDaBranch [deleta branch local]
b) git push origin --delete nomeDaBranch [deleta branch remota]
